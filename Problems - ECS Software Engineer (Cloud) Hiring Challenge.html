ECS Challenge

[Problem Statement]

You are a developer for Sigma, Ltd. Sigma is online bookstore company, and you are responsible for development of a website and backend infrastructure for a bookstore. 

 

Sigma, Ltd. would like you to develop backend resources to support online book store website as mentioned in the requirements section.

 

Requirements and deliverables from mandatory section are must to attempt and desirable section is optional and will qualify for bonus points if delivered. 

 

Below steps outlines overall development journey you will be following during the course of this challenge.

 

  • Setup Development environment as per Prerequisites section

  • Import provided sample data in local DynamoDB table.

  • Develop API endpoints using Python flask on development machine as per the requirements.

  • Create unit tests to cover the Acceptance Criteria for each API endpoint.

  • Upload the deliverables on the location mentioned in Guide section.


 

[Prerequisites]

Download and install pre-requisites mentioned below and start the local DynamoDB server.

 

Setup

Download and install Instructions

OpenJDK

Download and install setup from any of the link link1  or link2

Note: Allow the installer to set JAVA_HOME

Local DynamoDB

Download resources and setup local dynamoDB using this link.

NOSQL Workbench (Optional)

Download and install NOSQL Workbench on your local machine using this link.

 

Change directory location in console to the location where Local DynamoDB resources are extracted and start local DynamoDB using command: 


java -Djava.library.path=./DynamoDBLocal_lib -jar DynamoDBLocal.jar -sharedDb


 

 

 

[Requirements]

[Mandatory Requirements]

  • Setup a local DynamoDB table for maintaining book inventory. DynamoDB table can be setup with books sample inventory by following instructions.txt from import_books_inventory.zip

 

 

  • Develop API endpoints with python flask for following operations and develop respective lambda functions

API Routes

Details

get_books(filter, start_page, page_size)

Get list of books based on the specified filter

Ex.  Filter: “name=Harry Potter”,   Start_page = 2, page_size=20.
This example filter should return all “Harry Potter” books split into a list size of 20, starting with second chunk (skipping first 20 results)

get_book(id)

Get the specified book with bookID

add_book(params)

Add the specified book

update_book(id, params)

Update the specified book

get_favourite(user_id)

Get list of favourites marked books list for user

add_favourite(user_id, book_id)

Add a book to users favorite list

remove_favourite(user_id, book_id)

Remove book from users favorite list

 

[Desirable Requirements] 

  • Develop a frontend for a bookstore to consume data from API endpoints 

  • Configure token-based authentication for API endpoints

  • Create a new efficient DynamoDB schema for existing solution by following AWS recommended Single-table design for DynamoDB


 

[Deliverables]

[Mandatory Deliverables]

  • Functioning API routes as per the requirement specifications.

  • Unit tests with code coverage for all API endpoints (pytest or any other suitable framework), with a test evidence snap-shot generated. (in excel, json or any other suitable format)

  • Once completed, participants are expected to zip source code for all mentioned deliverables and upload to the mentioned location by following naming conventions mentioned in guide section.

 

[Desirable Deliverables]

  • Source code for frontend with brief document if required

  • API documentation for details on endpoints, return codes, parameters, return payload etc.

  • Singe table design DynamoDB Schema (in excel or export from NoSQL Workbench)

 

[Guide]

  • Tech Stack:  Python, Flask, DynamoDB (Local)

  • Development Environment: All Development will happen on your development machine locally.

  • Python standards: Please refer PEP8 convention

  • Deliverables Naming conventions: parent folder name should be named as {candidate-name} - {PhoneNumber} and child resources should be arranged as mentioned in the screenshot.

  • Upload location: Upload by clicking Upload File button below
